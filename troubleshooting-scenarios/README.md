# Troubleshooting Scenarios

Break/fix exercises and diagnostic walkthroughs — the core skill of every IT help desk job.

## Why This Folder Matters

Troubleshooting is the #1 thing you do in IT help desk. This folder documents real scenarios where something was deliberately broken or misconfigured, and the step-by-step process used to diagnose and fix it.

## Methodology Used

For every scenario, I follow this structure:
1. **Identify the problem** — what symptoms are present?
2. **Establish a theory** — what are the likely causes?
3. **Test the theory** — what steps confirm or rule it out?
4. **Plan of action** — what is the fix?
5. **Implement the fix** — carry it out
6. **Verify full functionality** — did it work? any side effects?
7. **Document** — write it up here

## Scenario Categories

- Network connectivity issues
- OS boot and startup failures
- Application errors and crashes
- User account and permission issues
- Hardware recognition problems
- Slow performance diagnosis
